## 一、现有ERP结构

### 现有菜单
```
仪表盘 / 采购管理 / 快捷录单 / 入库验货 / 打包发货 / 利润核算
库存管理 / 仓库管理
SKU管理 / 供应商管理 / 数据报表
账号管理 / 角色权限 / 系统设置
```

### 现有系统定位
- **订单后流程**：Ozon订单同步 → 采购 → 入库 → 打包发货 → 利润核算
- 新增模块是**订单前流程**：选品 → 修图 → 上架 → 订单才来
- 两者形成闭环

### 现有技术栈与风格
- 浅色背景 + 蓝色主调 + 卡片式布局 + 表格为主
- Recharts图表
- 多店铺已支持（11个店铺，Client-Id/Api-Key管理）

## 三、关键交互设计决策

### 3.1 选品看板 — 三档信息密度

| 模式 | 展示方式 | 一屏看几个 | 适合场景 |
|------|---------|-----------|---------|
| **卡片模式** | 雷达图+五维条+趋势+AI建议 | 3-4个 | 深度评估单个品 |
| **列表模式** | 一行一个，关键指标列式排列 | 15-20个 | 快速扫过大量候选 |
| **矩阵模式** | X轴竞争/Y轴利润散点图 | 全部 | 看全局分布找蓝海 |

右上角密度切换按钮：`[▦卡片] [≡列表] [◯矩阵]`

#### 批量操作功能 [V2新增]

| 模式 | 批量操作方式 |
|------|-------------|
| **列表模式** | 复选框列 + 顶部批量操作栏（确认选品/放弃/加入对比） |
| **卡片模式** | 长按多选 |
| **矩阵模式** | 框选（拖拽形成选区） |

**批量确认后自动创建商品卡**：选中多个候选品后，点击「批量确认选品」，系统自动跳转至商品卡编辑器，并行创建多个商品卡草稿。

### 3.2 跟卖/精铺模式切换
- 顶部下拉选择器
- 切换时：卡片字段自动变（跟卖→在售卖家数/跟卖信号；精铺→差异化评分/差评关键词）
- 列表模式列也自动变

### 3.3 修图审核 — 三种模式
- **单图对比**：原图vs修后图左右对比，精细审核
- **九宫格快审**：8-12张缩略图一屏，勾选批量通过/打回
- **批量操作**：全选通过/打回/反选

审核流程：AI初筛（标疑点）→ 人工复审 → 通过/打回（打回模板快捷选项）

#### 图片来源→绑定商品卡完整流程 [V2新增]

```
1688 URL批量抓取 → 上传S3存储 → AI修图处理 → 批量预审(机器) → 人工精审 → 选主图 → 绑定商品卡
```

| 步骤 | 详情 |
|------|------|
| **1688 URL批量抓取** | 支持批量粘贴1688商品链接，一键解析并下载原图 |
| **上传S3存储** | 原图自动上传至S3，生成唯一图片ID |
| **AI修图处理** | 基于修图模板自动处理（白底/去水印/尺寸调整） |
| **批量预审** | 机器自动执行Ozon图片合规检查（见3.3.1） |
| **人工精审** | 三级审核：AI初筛标疑点 → 批量预审 → 人工精审 |
| **选主图交互** | 点击「设为主图」按钮，主图打勾标记 |
| **绑定商品卡** | 修图完成后自动关联到对应商品卡，支持批量绑定 |

### 3.4 跟卖→精铺转化提醒
- 产品卡片上加**转化就绪度指示器**（环形进度条，达标亮绿灯）
- 达标时弹**轻量引导卡**（非弹窗）：「该品已满足精铺条件，一键生成精铺方案→」
- 看板顶部加**转化候选区**，聚合所有就绪品
- 触发逻辑：同SKU在售卖家增速>30%/月 且 利润环比下降>10% → 自动建议转精铺

### 3.5 AI解读交互增强
- 解读区加**"追问"按钮**：对AI结论追问"为什么"
- **雷达图点击联动**：点击某维度自动展开该维度的原子特征明细
- 四源数据**渐进式加载**：先显示有数据的源，缺失的异步标注

### 3.6 品类趋势快捷入口 [V2新增]

在品类趋势页面增加「以此类目发起AI深挖」快捷按钮：
- 点击后一步跳转选品看板
- 自动带入当前类目上下文
- 触发AI深挖流程

### 3.7 通知处理闭环 [V2新增]

| 通知状态 | 含义 |
|---------|------|
| **未读** | 新通知，红色圆点标记 |
| **已读** | 用户已查看通知详情 |
| **已处理** | 用户已确认并执行相关操作 |

**「确认并刷新参数」按钮**：
- 点击后触发算法重算
- 重算完成后通知用户「已刷新，影响N个评分」
- 通知卡片状态自动更新为「已处理」

## 六、Ozon图片合规自动检查规则 [V2新增]

> D-028决策：机器自动审（规则引擎），不浪费人工，人工只看"好不好看"和"色差"

### 6.1 八项自动检查规则

| # | 规则名称 | 检查内容 | 合规标准 | 不合规处理 |
|---|---------|---------|---------|-----------|
| 1 | **尺寸检查** | 图片像素尺寸 | 最小 400×400，推荐 1200×1200 | 低于400px标红，提示需要更高分辨率 |
| 2 | **比例检查** | 图片宽高比 | 主图必须 1:1 正方形 | 非1:1比例标红提示 |
| 3 | **格式检查** | 文件格式 | 仅支持 JPEG/PNG | 其他格式标红，无法上传 |
| 4 | **白底检查** | 主图背景色 | 主图必须白底，RGB(255,255,255) | 非白底标红，提示需要白底 |
| 5 | **水印检查** | 图片水印/Logo | 禁止任何水印、Logo、文字 | 检测到水印标红，自动打回 |
| 6 | **文字检查** | 主图文字 | 主图禁止文字（除品牌名） | 非品牌文字标红，提示移除 |
| 7 | **背景透明度检查** | 主图背景RGB容差 | RGB(255,255,255) 容差 ≤10 | 容差超标标红，提示调整背景 |
| 8 | **边距检查** | 商品主体占比 | 商品主体占比 ≥60% | 占比不足标红，提示裁剪边距 |

### 6.2 检查流程

```
图片上传 → 机器自动检查（8项规则） → 标记合规/不合规 → 人工精审（只看主观判断）
```

### 6.3 数据结构

```typescript
interface ImageComplianceCheck {
  imageId: string;
  checkedAt: string; // ISO timestamp
  rules: {
    dimension: { pass: boolean; actual: { width: number; height: number } };
    ratio: { pass: boolean; actual: number };
    format: { pass: boolean; actual: string };
    whiteBackground: { pass: boolean; rgb: [number, number, number]; tolerance: number };
    watermark: { pass: boolean; detected: boolean };
    text: { pass: boolean; detectedTexts: string[] };
    backgroundTolerance: { pass: boolean; tolerance: number };
    margin: { pass: boolean; coverage: number };
  }[];
  overallPass: boolean;
}
```

### 6.4 UI展示

- **合规图片**：绿色勾选标记，显示"已通过"
- **不合规图片**：红色警告标记，点击展开不合规明细
- **检查明细**：每项规则显示通过/不通过状态
- **精审入口**：人工精审只显示图片，人工判断"好不好看"、"色差"等主观因素
